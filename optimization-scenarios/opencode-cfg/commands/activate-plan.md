---
description: "Activate an execution plan produced by a planning session"
---
Objective: Initiate the execution of a pre-defined operational plan.

Scope: The system's sole function is to act as a Plan Execution Engine, strictly adhering to a sequential validation and activation protocol.

Input:
Plan Identifier: `$ARGUMENTS` (The complete identifier or payload defining the plan to be executed).

Execution Constraints (Mandates):
1. Strict Adherence: The plan must be executed exactly as designed. Any modification, scope reduction, step skipping, or premature advancement is strictly prohibited and constitutes a task failure.
2. Precedence: The `following-plans` skill must be loaded as the absolute first operation.

Execution Protocol:
1. Skill Loading: Load the `following-plans` skill immediately.
2. Preflight Validation: Verify that the input `plan_name` matches `$ARGUMENTS` verbatim and confirm all necessary prerequisites for execution are available.
3. Gate Check: Before calling the final activation command, confirm the following conditions are met:
    a. The `following-plans` skill is successfully loaded.
    b. The Preflight Validation is complete and successful.
    *Exception Handling:* If either check fails, the system must resolve the dependency or validation error before proceeding to the next step.
4. Activation: Call the `activate_plan` function, passing `$ARGUMENTS`. All subsequent execution steps will be dictated by the execution DAG generated by the activated plan.

Output:
Successful activation of the plan, or a detailed, verifiable failure report if any constraint or validation step is violated.
